SmartPerfetto: 11 层 Harness Engineering

从用户提问到分析报告的完整技术栈

INPUT ROUTING
01
场景分类
纯关键词匹配，12 场景，来自 .strategy.md frontmatter。零 LLM 调用，省 ~3500 tokens/轮
<1ms / 12 scenes
02
复杂度路由
Hard Rules (70%) + Haiku 分类 (30%)。简单问题走 Quick Path 秒回，复杂问题走 Full 深钻
Quick: 3-5s / Full: 30-90s
PROMPT ENGINEERING
03
动态 System Prompt
4-Tier 排序 (STATIC → TRACE → QUERY → DYNAMIC) 最大化前缀缓存命中。Token 预算 4500，超限从底部逐段丢弃
~80% cache hit rate
EXECUTION ENGINE
04
MCP 工具系统
20 工具 (9 常驻 + 11 条件)。Planning Gate 强制先想再做，REASONING_NUDGE 每步反思
20 tools / 60 max turns
05
SQL 纠错学习
Jaccard 相似度匹配 error→fix 对，跨会话持久化。30 天 TTL，每会话加载最近 10 条
30-day TTL / 10 pairs
TOKEN EFFICIENCY
06
Artifact Store
LRU 缓存 + 3 级 fetch (summary/rows/full)。Claude 看摘要 (~440 tokens)，前端看全量 DataEnvelope
~85% token savings
REAL-TIME STREAMING
07
SSE Bridge
200ms 缓冲窗口区分 thought vs answer。SDK stream → 语义化 SSE 事件 → 前端实时渲染
200ms buffer / 7 event types
DOMAIN KNOWLEDGE
08
YAML Skill 系统
160+ 声明式技能，L1-L4 四层结果。厂商覆写 (.override.yaml) 适配 8 大平台
160+ skills / 8 vendors
09
Sub-Agent 专家
3 个专家 (frame/system/startup)，并行深钻。120s 超时，bestEffort 模式不阻塞主分析
3 experts / 8 turns each
QUALITY ASSURANCE
10
三层验证
启发式 (regex) + Plan 遵从 + Haiku 独立审查。纠正循环最多 2 轮，拦截 ~30% 误诊
~30% false positives caught
LEARNING & MEMORY
11
跨会话学习
Pattern Memory (正向+负向) + SQL Error-Fix Pairs + Misdiagnosis Learning。越用越准
90-day TTL / 30 patterns
